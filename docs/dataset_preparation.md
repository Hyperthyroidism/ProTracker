# Dataset Preparation

This document explains how to prepare the Vessel-MOT dataset for ProTracker.

ProTracker is designed for UAV-based high-density vessel tracking. The dataset should contain videos, image frames, YOLO-format detection labels, and MOT-format tracking annotations.

## 1. Expected Dataset Structure

The dataset should be placed under:

```text
datasets/Vessel-MOT/
```

The recommended structure is:

```text
datasets/Vessel-MOT/
├── images/
│   ├── train/
│   ├── val/
│   └── test/
├── labels/
│   ├── train/
│   ├── val/
│   └── test/
├── videos/
│   ├── train/
│   ├── val/
│   └── test/
└── annotations/
    ├── train/
    ├── val/
    └── test/
```

## 2. Directory Description

### images

```text
datasets/Vessel-MOT/images/
```

This directory stores extracted video frames.

Example:

```text
datasets/Vessel-MOT/images/train/sequence_001/
├── 000001.jpg
├── 000002.jpg
├── 000003.jpg
└── ...
```

### labels

```text
datasets/Vessel-MOT/labels/
```

This directory stores YOLO-format labels.

YOLO-format labels are mainly used for training or checking the vessel detector.

Example:

```text
datasets/Vessel-MOT/labels/train/sequence_001/
├── 000001.txt
├── 000002.txt
├── 000003.txt
└── ...
```

Each YOLO label file follows this format:

```text
class_id x_center y_center width height
```

All coordinates are normalized to `[0, 1]`.

Example:

```text
0 0.5123 0.4388 0.0452 0.0317
```

### videos

```text
datasets/Vessel-MOT/videos/
```

This directory stores original UAV videos.

Example:

```text
datasets/Vessel-MOT/videos/test/
├── sequence_001.mp4
├── sequence_002.mp4
└── sequence_003.mp4
```

### annotations

```text
datasets/Vessel-MOT/annotations/
```

This directory stores MOT-format tracking annotations.

Example:

```text
datasets/Vessel-MOT/annotations/test/
├── sequence_001.txt
├── sequence_002.txt
└── sequence_003.txt
```

Each annotation file follows the MOT format:

```text
frame_id, track_id, x, y, width, height, confidence, class_id, visibility
```

Example:

```text
1,1,512.400000,236.700000,42.500000,18.300000,1.000000,0,1.000000
```

## 3. Create Dataset Directories

You can automatically create the required dataset directories by running:

```bash
python scripts/prepare_vesselmot.py \
  --root datasets/Vessel-MOT \
  --create
```

You can also check the dataset structure:

```bash
python scripts/prepare_vesselmot.py \
  --root datasets/Vessel-MOT \
  --check
```

To print a simple dataset summary:

```bash
python scripts/prepare_vesselmot.py \
  --root datasets/Vessel-MOT \
  --summary
```

Or run all operations together:

```bash
python scripts/prepare_vesselmot.py \
  --root datasets/Vessel-MOT \
  --create \
  --check \
  --summary
```

## 4. Convert YOLO Labels to MOT Format

If only YOLO-format labels are available, they can be converted to MOT-style annotation files using:

```bash
python scripts/convert_yolo_to_mot.py \
  --label-dir datasets/Vessel-MOT/labels/test/sequence_001 \
  --image-dir datasets/Vessel-MOT/images/test/sequence_001 \
  --output datasets/Vessel-MOT/annotations/test/sequence_001.txt
```

However, YOLO-format labels usually do not contain identity information. Therefore, the converted MOT file only provides a temporary identity assignment. For strict multi-object tracking evaluation, manually checked tracking IDs are required.

## 5. Dataset Configuration

The dataset path is defined in the configuration files.

For training:

```text
configs/train/ProTracker.yaml
```

For testing:

```text
configs/test/ProTracker.yaml
```

Example:

```yaml
dataset:
  name: Vessel-MOT
  root: datasets/Vessel-MOT
  image_dir: datasets/Vessel-MOT/images
  video_dir: datasets/Vessel-MOT/videos/test
  annotation_dir: datasets/Vessel-MOT/annotations
  class_names:
    - vessel
```

If your dataset is placed in another directory, modify these paths accordingly.

## 6. Testing Data

For testing, ProTracker mainly uses videos under:

```text
datasets/Vessel-MOT/videos/test/
```

The corresponding ground-truth annotations should be placed under:

```text
datasets/Vessel-MOT/annotations/test/
```

The file names should match.

Example:

```text
datasets/Vessel-MOT/videos/test/sequence_001.mp4
datasets/Vessel-MOT/annotations/test/sequence_001.txt
```

The prediction result generated by ProTracker will be saved as:

```text
results/test/sequence_001/sequence_001.txt
```

or:

```text
results/test/sequence_001.txt
```

depending on the output setting.

## 7. Training Data

For training the graph-based cascaded prompt optimizer, the training split should include images, labels, and annotations.

Recommended structure:

```text
datasets/Vessel-MOT/images/train/
datasets/Vessel-MOT/labels/train/
datasets/Vessel-MOT/annotations/train/
```

The training process mainly uses:

```text
image frames
detection boxes
tracking identities
temporal associations
```

These data are used to construct graph nodes, graph edges, and edge labels.

## 8. Notes on Large Files

The complete Vessel-MOT dataset is not included in this repository because video files and annotations can be large.

The following directories are ignored by `.gitignore`:

```text
datasets/
data/
Vessel-MOT/
```

Therefore, users should prepare the dataset manually.

## 9. Minimal Demo Dataset

For a quick code test, a minimal dataset can contain only one short video:

```text
assets/demo.mp4
```

Then run:

```bash
python run.py \
  --config configs/test/ProTracker.yaml \
  --video assets/demo.mp4 \
  --output results/demo
```

This demo mode does not require the full Vessel-MOT dataset.
